# Roadmap

This roadmap is intentionally conservative. Items here are planned directions,
not guarantees.

## Near Term

- Improve precision for existing contract patterns.
- Keep the CLI fail-closed for staged, untracked, binary, and unmerged inputs.
- Strengthen CI examples for trusted baseline checks.
- Improve documentation around manual contracts and intentional evolution.

## Later

- Expand review-friendly reporting formats.
- Evaluate additional TypeScript and JavaScript semantic patterns.
- Explore integrations that preserve Drift's local-first, no-cloud-required
  security model.
